qHTS client is the primary way to retrieve compound screening data.

Data can be retrieved based on a specific list of assay protocols, compounds, or potency or efficacy of the compound response.

It can be used to compare responses of a given compound across many different assays.


Software Requirements

The CR Client requires Java to run. Java is routinely available on most windows and Mac workstations and laptops. The CR Client launches by hitting a URL link and downloads the application and launches it using a technology called Java Web Start (JWS). Security warnings will warn about trusting the software. We don’t have an officially signed Java Archive (JAR file) so just accept the application when prompted. The client extracts data from our internal databases so the client will only function when connected to our internal NCATS network, either when working on-site or via a Virtual Private Network (VPN) connection or Remote Desktop Session (RDS).

Launching the Client

Use the CR Client Launch URL to start the application (http://qhts/ws/cr_client.jnlp). You can bookmark this url or add it to a bookmark toolbar for easy access. Initially you will be prompted on how to handle the ‘jnlp’ launch file. Select to ‘Open’ the file using Java Web Start Launcher. Next you will be prompted or warned about security. Java webstart downloads the program in a compressed file and launches the program on your machine. Select to ‘Run’ the application.
